# Asset Guide

## Purpose

Define how static assets are stored and referenced.

## Locations

- public/: primary location for static assets.
- Group folders by domain area (football, gaming, space, etc.).

## Naming Rules

- Use lowercase names.
- Use hyphens for multi-word files.
- Avoid generic names like image1.png.

## Optimization

- Compress large images before adding.
- Prefer modern formats when appropriate.
- Remove unused assets during cleanup cycles.

## Validation

- [ ] Asset paths resolve from consuming routes/components.
- [ ] No broken image links.
- [ ] Asset naming follows conventions.
