# Performance Checklist

Use before major releases or after adding heavy content.

## Page Weight

- [ ] Large images optimized.
- [ ] Unused assets removed.
- [ ] No unnecessary client-side scripts.

## Rendering

- [ ] Critical routes render quickly on first load.
- [ ] Expensive component trees reviewed.
- [ ] Avoid unnecessary re-renders in interactive components.

## UX

- [ ] Layout shift is minimal.
- [ ] Content is readable while assets load.
- [ ] Mobile experience remains smooth.

## Follow-Up

Record major performance findings in DECISIONS.md when architectural
changes are made.
