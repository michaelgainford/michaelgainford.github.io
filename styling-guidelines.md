# Styling Guidelines

## Stack

- Tailwind CSS for utility-first styling.
- Global styles in app/globals.css.

## Principles

- Prefer reusable patterns over one-off class strings.
- Keep spacing scale consistent.
- Favor semantic HTML with utility classes for layout and visuals.
- Minimize deeply nested wrappers when possible.

## Responsiveness

- Build mobile-first.
- Validate key breakpoints before merging.
- Avoid fixed widths unless required.

## Theming and Visual Consistency

- Keep typography, spacing, and color usage consistent across routes.
- Reuse shared classes and component styles where practical.

## CSS and Tailwind Hygiene

- Remove unused styles.
- Keep class names readable and logically grouped.
- Avoid inline style duplication if a utility or shared style can be used.
